1980

Naar HTML

Tim Berners-Lee, een backend developer van Engeland, startte in 1980 bij CERN. Hij ondervond moeilijkheden tijdens het opslaan en het sorteren van informatie. De software die daarbij uitkwam was ENQUIRE. Berners-Lee schreef een proposal naar zijn supervisor om een nieuwere manier te voor het managen van informatie. Hierbij kreeg hij als antwoord: “vaag, maar spannend”. Dit werd uiteindelijk het World Wide Web.

1981

Hyperspace

Het web maakt gebruik van HTTP, HyperText Transfer Protocol. Het verzend and krijgt data. De data is uniek aan iedere url waar het aan is gelinkt. Meeste van deze urls zijn opgemaakt uit HTML teksten. Het HREF attribuut laat de gebruiker van een URL naar een andere URL gaan. Dit creëert een link tussen een pagina naar de ander. Deze linkjes maken het web een doorslaggevende opslagruimte en ontvangsysteem tot een hypertext systeem. Ted Nelson, een andere backend developer bij CERN, heeft de term hypertest ontwikkelt. Hij noemde zijn systeem Xanadu.

1984

Line Mode Browser

Om de kracht van het web te demonstreren, moest een cross platform browser gemaakt worden. Eentje dat iedereen kan installeren en gebruiken, maakt niet uit welk opererend systeem iemand zou gebruiken. Deze taak werd gegeven aan Nicola Pellow, die nog een student was. Zij zette het Line Mode Browser op. Het was simpel, maar werkte heel goed. Het had niet dezelfde interactiviteit als het World Wide Web browser, maar het kon in feite op elk type machine en door iedereen gebruikt worden.

1991

Visuele oplossingen

Håkon Wium Lie werkte in dezelfde tijd met Tim bij CERN. Hij merkte al snel de potentie voor de taal, HTML, die voor de World Wide Web gebruikt werd. Hij realiseerde zich ook dat de kracht van de taal snel in gevaar kwam te staan door visuele factoren. Håkon stelde voor om een nieuw formaat op te zetten om de presentatie van de HTML documenten te beschrijven. Dit werd de nieuwe taal: Cascading Style Sheets. Samen met een Nederlandse programmeur, Bert Bos, zetten zij een syntax op die krachtig genoeg zou zijn om de visuele eisen van designers kan ondersteunen en ook makkelijk te leren is.

1996

Killer Websites

David Spiegel schreef het boek Creating Killer Websites, waarin hij meerdere technieken beschreef voor mooie designs die opgezet worden door HTML. Een van die technieken was een transparante GIF, die pixel per pixel in een grootte komt te staan. Als dit binnen een pagina werd geplaatst als een IMG element, dan werden precieze values gegeven in de WIDTH en HEIGHT attributen. Designers konden op deze manier controle houden over het aantal witruimte tussen hun designs.

2007

Steve's mobieltje

In 2007 bracht Steve Jobs het mobieltje uit dat de gebruiker ook toegang gaf tot het World Wide Web. Weinig mobiele telefoons konden een HTML aan, sommige konden een mobiel vriendelijke website geen in een WML formaat. Vanwege competitie onder de telefoons, gaven steeds meer telefoontjes toegang tot het internetten. Hierdoor kwamen nieuwe onderzoekspunten in de vergaderingen voor webdesign, zoals ‘Hoe weten we wanneer men een computer of een telefoon gebruikt?’ Ze zagen mobiele telefoons en computers niet meer als één apparaat, maar twee verschillende apparaten. Waardoor designers voor een website niet één, maar twee ontwerpen moesten maken voor een website.

2010

Één website

De markt verkocht na 2007 veel verschillende apparaten met verschillende schermgroottes dat het ontwerpen voor een website steeds lastiger begon te worden. In april 2010 gaf Ethan Marcotte een lezing bij An Event Apart in Seattle. Een bijeenkomst voor mensen die websites maken. Hij kwam met het voorstel van responsive design. Een idee dat gebouwen kunnen veranderen en aanpassen aan eisen van mensen die gebruik maken van het gebouw. Dit, zoals hij uitlegde, kon een manier zijn naar het maken van websites.

2012

Mobile First

Stephanie en Bryan Rieger kwamen met het mobiele-first responsie design aanpak: “De ontbreking aan een media query is jouw eerste media query.” In deze context gaat mobile first niet perse over de aandacht naar mobieltjes, maar op de prioriteit van content. Vanwege een klein scherm moeten alleen de belangrijkste onderwerpen aan bot komen. Hierdoor worden aannames vermindert. Jarenlang was een website een creatie van een webdesigner. De gebruiker had geen zeggenschap in de flow. Hij moest zich aanpassen aan de eisen voor een bepaalde schermgrootte, snelheid in netwerk of zwaarte van de visuele punten. Nu kan webdesign een conversatie starten tussen designer en gebruiker. Webdesign kan het onderliggende principe reflecteren voor het web zelf.
